Generating an access key pair

An access key pair is used to authenticate third-party applications that can access the archival storage. For example, you need a key pair to configure an S3-compatible client. For more information, see Accessing archival storage via third-party clients.

An access key pair consists of the following components:

Access key ID
Secret access key

You can only use the access key pair with a specific endpoint URL. For example: s3-us-central-2.acronis.storage. This URL depends on the data center location for your account, and you cannot change it.

To generate an access key pair

Initial pair
New pair
In the Cyber Protect console, go to Backup storage > Archival storage.

Click Generate access key pair.

You can see, copy, or download the access key ID, secret access key, and endpoint URL only when you generate the access key pair. If you lose or forget this data, you cannot retrieve it, and you must generate a new key pair.
Select the I understand and agree checkbox.
Click Generate.

Copy the access key pair and endpoint URL or download them as a CSV file, and then click Close.

Ensure that you store this data securely.
