# Editing or deleting a script

Managing workloads in the Cyber Protect console > Cyber Scripting > Scripts > Editing or deleting a script
Editing or deleting a script

Depending on your user role, you can perform different operations with scripts and scripting plans. For more information about the roles, see User roles and Cyber Scripting rights.

To edit a script

In Script repository, go to My Scripts, and then find the script that you want to edit.

Click the ellipsis (...) next to the script name, and then click Edit.

Edit the script, and then click Save.

[If you edit a script that is used by a scripting plan] Confirm your choice by clicking Save script.

The latest version of the script will be used next time the scripting plan runs.
Script versions

A new version of the script is created if you edit any of the following script attributes:

script body
script name
description
script language
credentials
arguments

If you change other attributes, your edits will be added to the current script version. To learn more about versions and how to compare them, refer to Comparing script versions.

The script status is updated only when you modify the value in the Status field. Only administrators with the Cyber administrator role can change a script status.

To delete a script

In Script repository, go to My Scripts, and then find the script that you want to delete.

Click the ellipsis (...) next to the script name, and then click Delete.

Click Delete.

[If you want to delete a script that is used by a scripting plan] Confirm your choice by clicking Save script.

Scripting plans that use the deleted script will fail to run.
