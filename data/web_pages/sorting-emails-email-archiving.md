# Sorting emails

Managing the backup and recovery of workloads and files > Email archiving > Search queries > Sorting emails
Sorting emails

You can sort emails at the following levels:

The email archive (All emails)
A user mailbox
The Incoming or Outgoing folder in a user mailbox

To sort emails

Log in to the Cyber Protect console as administrator.
Go to Backup storage > Backups > Cloud application backups.
[To sort emails in an archive in immutable storage] Click Show deleted.
Select an email archive, and then click Browse archive.
[If a privacy warning is shown] In the Privacy warning dialog, click Proceed.
In the Encryption password dialog, specify the password, and then click Proceed.

Select the level at which you want to sort emails.

You can select All emails, a user mailbox, or the Incoming or Outgoing folder in a user mailbox.

To search for the mailbox of a specific user, in the left-column search field, type at least three characters of the user name, and then press Enter.

On the main screen, click the sort button.

Select the sort criterion and sort order.

Sort criteria	Sort order
Date	Chronological: Newest first, Oldest first
Subject	Alphabetical: A–Z, Z-A
Recipient

Alphabetical: A–Z, Z-A

Emails with multiple recipients are sorted according to the first recipient's name.


Sender

Alphabetical: A–Z, Z-A


Attachments

With attachments on top (newest to oldest), Without attachments on top (newest to oldest)


Relevance

Only available when you perform a search or use a saved search query.

The relevancy algorithm emphasizes match quality over quantity by implementing a multi-tiered scoring system. Exact matches receive the highest weight and follow a max-score rule - one exact match can outweigh many partial matches. Unique term diversity is prioritized next, meaning that the number of exact phrase matches is more important than match frequency of individual terms, which helps reduce the influence of keyword repetition. Email timestamp functions as a tiebreaker when items have identical match quality. Additionally, content length is normalized to ensure that longer emails do not automatically outrank concise, highly relevant ones.
