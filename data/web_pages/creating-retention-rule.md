# Creating a retention rule

Managing the backup and recovery of workloads and files > Email archiving > Retention rules > Creating a retention rule
Creating a retention rule

You can create a retention rule from the email archive or from the Management > Archiving plans tab.

Retention rules that are created from the email archive can use only one search query and one email archive.

Retention rules that are created from the Management > Archiving plans tab can use multiple search queries and multiple email archives.

To create a retention rule

From the email archive
From Management > Archiving plans
Run a search query on the email archive, as described in Running a saved search query.

Above the list of matching emails, click Apply retention rule.

[If there are existing retention rules] In the Apply retention rule to search query window, click Create retention rule.

To change the retention rule name, click the pencil icon, specify a new name, and then click OK.

In Retention period, select an option.

[If you select By email age] Specify the time period and select the time unit.

Click Save.

For example, you can specify 1 year, 2 months, or 300 days. Emails that are older than the selected period will be deleted from the archive. The email age is calculated based on when each email was sent or received.

In Encryption password for archives, specify the encryption password for the email archive, and then click Proceed.

Click Create.

As a result, a retention rule is created.

Emails that match any of the search queries in this retention rule will be deleted when the retention period ends.

If an email matches more than one retention rule, the rule with the longest retention period is applied.
