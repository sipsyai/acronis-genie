# Failing back

Managing the backup and recovery of workloads and files > Special operations with virtual machines > Working in VMware vSphere > Failing back
Failing back

To failback from a replica

Select a replica that is in the failover state.
Click Replica actions.

Click Failback from replica.

The software automatically selects the original machine as the target machine.

[Optional] Click Target machine, and then do the following:

Select whether to failback to a new or existing machine.
Select the ESXi host and specify the new machine name, or select an existing machine.
Click OK.

[Optional] When failing back to a new machine, you can also do the following:

Click Datastore to select the datastore for the virtual machine.
Click VM settings to change the memory size, the number of processors, and the network connections of the virtual machine.
[Optional] Click Recovery options to modify the failback options.
Click Start recovery.
Confirm your decision.
