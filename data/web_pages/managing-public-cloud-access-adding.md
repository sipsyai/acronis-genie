# Adding access to a public cloud connection

Managing the backup and recovery of workloads and files > Backing up workloads to public clouds > Managing public cloud account access > Adding access to a public cloud connection
Adding access to a public cloud connection
Buckets in special GovCloud regions (for example, https://aws.amazon.com/govcloud-us/, https://learn.microsoft.com/en-us/azure/azure-government/regions, or other endpoint URLs from storage vendors supporting the GovCloud concept) are not supported as backup locations.

After adding a public cloud connection for Amazon S3 or Microsoft Azure storage in the Cyber Protect console, you can securely access your cloud resources and directly back up workloads to the relevant public cloud storage.

After you add a location, you cannot remove the access to the public cloud. To remove the access, you must delete the location. See Deleting a public cloud connection.

For more information about adding access for Microsoft Azure cloud storage, see Managing access to Microsoft Azure subscriptions.

To add access to a public cloud connection

In the Cyber Protect console, go to Infrastructure > Public clouds.

Click Add, and select one of the following options:

Amazon S3
S3 compatible
Impossible Cloud

Click Connect.

The connection process starts, and may take several minutes. When finished, the connection is added to the list of public clouds.

To renew the annual access certificate for the connection, see Renewing access to a public cloud connection.

To remove access to the connection, see Deleting a public cloud connection.
