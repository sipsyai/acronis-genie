# Cloning a workflow

Managing and assigning automation workflows > Managing your workflows > Cloning a workflow
Cloning a workflow

You can clone any existing workflow that is currently set as Enabled.

When cloning a workflow, a full copy of the default version of the workflow is cloned, including all triggers, conditions, and actions.

When logged in as a Company administrator, Administrator, or Cyber administrator, you can clone any workflow, including workflows in Draft status.

To clone a workflow

In the Cyber Protect console, go to Management > Workflows.

Ensure that the workflow you want to clone is set with the Enabled status.

In the relevant workflow row, click the ellipsis (...) icon, and then select Clone.

The cloned workflow is saved in Draft status, and with the name Copied + Old workflow name. For example, Copied Alerts workflow. To change the name of the cloned workflow, open the workflow, and update the name in the Settings tab. See Updating a workflow.
