How URL filtering works

When URL filtering is enabled, it filters all the network traffic that goes into the machine, including browser activity and any other network activity happening in the background.

For example, when a user enters a URL in a browser, the Interceptor gets the link and sends it to the protection agent. The agent gets the URL, parses it, and then checks the verdict. The Interceptor redirects the user to the page with the message with available actions to manually proceed to the requested page.
