# Changing the registration of a workload

Installing and deploying Cyber Protection agents > Registration of workloads > Changing the registration of a workload
Changing the registration of a workload

You can change the registration of a workload by registering it in a new tenant or under a new user account.

When you change the registration of a workload, all protection plans that are applied to it will be revoked. To continue protecting the workload, apply a new protection plan to it.

If you register the workload in a new tenant, the workload will lose access to the backups in the cloud storage of the original tenant. The backups in non-cloud storages will remain accessible.

To change the registration of a workload

By using the command-line interface
By using the graphical user interface
Unregister the protection agent, as described in Unregistering workloads.
Register the protection agent in the new tenant or under the new user account, as described in Registering workloads by using user credentials or in Registering workloads by using a registration token.
