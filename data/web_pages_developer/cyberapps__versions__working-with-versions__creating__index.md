# Creating a new Version

Creating a new Version

Since the CyberApp functionality can change over time, you can create multiple CyberApp Versions.


Note

When you create a CyberApp Version, the initial state is .
The maximum number of Versions in the  state is 10.



Note
You can also create a new Version as a copy of an existing Version by duplicating one of the CyberApp Versions or importing a Version package from another CyberApp.


To create a new Version

Open the CyberApp Version list.
Click .
Enter a unique version number in the Version field.

The version number helps you manage multiple CyberApp Versions.
We recommend incrementing the minor number (For example, 1.1 -> 1.2) for lesser changes, and incrementing the major number (For example, 1.5 -> 2.0) only for more significant changes.


[Optional] Enter a Comment.

This only appears in the Version list (as the Description column entry).
It helps identify Versions and is used to search the Version list.


Set the CyberApp enablement method.


Note
You can change the enablement method in the General Settings section if you edit the CyberApp Version.



Click Add.



The new CyberApp Version is opened on the General Settings section.
