            There is the Acronis Cyber Protect appliance (all in one), but it is CentOS based, and as this OS is end of support.
From the release notes of Cyber Protect 16 we get
Switch from CentOS 7 to Rocky Linux for Acronis Management Server in form of Docker container image
Acronis Management Server component delivered in Docker container image format, is now based on Rocky Linux instead of CentOS, due to CentOS deprecation. Upgrade scenarios from previous CentOS-based versions, are supported.

Does this mean that there will be no new release of the appliance with a news OS base?
With the appliance choice, Acronis offers the customer an easy going start, and a base that is more easily supportable, agree?
When the customer chooses Docker network separation of the backup infrastructure will not so easy to be achieveable.
So, shouldn't Acronis decide to make up a new appliance with a modern OS base?
