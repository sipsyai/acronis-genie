            Hello,
In R1Soft we had the possibility to lock many recovery points.
Let's say our backup plan has 30 days backups. But customer would like us to hold the backup for 5 extra days.
So instead of changing the plan as that would change all agents/servers using the same plan, we could lock 5 extra recovery points just for that agent.
All locked recovery points will never get deleted until they are unlocked.

            Hello Everyone,
last year, we've introduced an new feature Immutable backups for Acronis Cyber Protect Cloud 21.12 release
Customers’ backups will not be lost and backed up data can be easily recovered after a malware attack, or an accidental or malicious deletion of backups.
When immutable backup functionality is enabled, data deletion on Acronis Cyber Protect Cloud storage (Acronis-hosted or partner-hosted) is delayed for a configurable number of days (set in Acronis Cyber Protect). This enables partners to recover data from deleted backups via Acronis Cyber Protect console without having to contact Acronis support.
Here you can read more on this feature https://kb.acronis.com/content/69814 
Let me know, if that covers your scenario? Do you have any additional requirements?
