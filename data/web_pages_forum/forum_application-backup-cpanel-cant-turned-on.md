            Hello Emanuel,
thank you for your posting! As it turns out, the application-aware backup feature for cPanel is not in a production-ready state and cannot be enabled at the moment. Currently, the backup operation can be performed from the cPanel side, please follow the article: https://www.acronis.com/en-us/support/documentation/WHMCPanel/#enabling-backup-for-server.html

            Hello!
Kindly review the following statement from our team:
- The 'cPanel’ option in the ‘Application backup’ section stands for collecting cPanel metadata via the protection agent's code in the standard backup flow rather than running scripts at the pre/post data capture step of a protection plan. The new approach is expected to be a more secure, faster, and more reliable option.
- The feature and the toggle will be made available with the release of cPanel plugin version 1.9. Release availability will be announced through the dedicated mailing list ( https://promo.acronis.com/hsp-mailing-subscription.html ).
- Enabling the toggle won't enable the ability to restore and download hosting panel objects (accounts, websites, databases, mailboxes, etc.) from Acronis Cyber Protect Cloud Console but will only change the approach towards application metadata collection.
